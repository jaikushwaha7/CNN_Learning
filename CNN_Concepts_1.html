üéØ Understanding Padding

Padding adds extra pixels around the input to control the output size and preserve edge information.

Output Height = Input Height + 2 √ó Padding - Kernel Height + 1
Output Width = Input Width + 2 √ó Padding - Kernel Width + 1

üìè Common Padding Types

Valid Padding (No Padding):
Output size = Input size - Kernel size + 1
Output shrinks with each layer
Same Padding:
Padding = (Kernel size - 1) / 2
Output size = Input size
Full Padding:
Padding = Kernel size - 1
Output size = Input size + Kernel size - 1

üîç Why Use Padding?

  • Preserve spatial dimensions
  • Maintain edge information
  • Enable deeper networks
  • Better feature extraction at borders

‚ö° Stride Effects

Stride = 1: Maximum overlap, preserves all information
Stride = 2: 50% overlap, reduces output size by ~50%
Stride > Kernel Size: Gaps in coverage, information loss

üéØ When to Use Different Strides

  • Stride = 1: Feature extraction layers
  • Stride = 2: Downsampling while convolving
  • Stride > 2: Aggressive downsampling (rare)

üìä Max vs Average Pooling

Max Pooling:
‚Ä¢ Selects maximum value in each region
‚Ä¢ Preserves strongest activations
‚Ä¢ Better for feature detection
‚Ä¢ More translation invariant
Average Pooling:
‚Ä¢ Computes mean of each region
‚Ä¢ Smooths the feature map
‚Ä¢ Reduces noise
‚Ä¢ Less common in modern architectures

üîß Pooling Guidelines

  • Common: 2√ó2 pooling with stride 2
  • Effect: Reduces spatial size by 50%
  • Position: After activation functions
  • Channels: Applied independently to each channel
